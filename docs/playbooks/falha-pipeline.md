# Playbook — Falha de Pipeline

## Passo 1: Classificar a falha
- Código?
- Teste?
- Ambiente?
- Dependência externa?

## Passo 2: Avaliar impacto
- Fluxo crítico afetado?
- Falha recorrente?

## Passo 3: Decisão
- Corrigir e reexecutar
- Ignorar com evidência
- Bloquear release

## Evidências mínimas
- Link do pipeline
- Log relevante
- Justificativa registrada

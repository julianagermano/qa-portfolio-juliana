# Estratégia de Testes de Dados

A estratégia de testes de dados é orientada a risco e contexto do produto.

Princípios:
- QA testa o sistema via API ou fluxo do produto
- O banco é utilizado como evidência
- Não se testa banco isoladamente

Tipos de validação:
- persistência correta
- consistência entre entidades
- estado final após falhas
- dados duplicados ou órfãos
